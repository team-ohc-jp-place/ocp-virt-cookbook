OpenShift Virtualization cookbook

Introduction

A practical field guide for starting and deploying workloads on OpenShift Virtualization.

This cookbook provides bite-sized recipes, example manifests, and troubleshooting notes for those who are just starting with OpenShift. It is designed as a hands-on complement to the official documentation and training course - facilitating real-world implementation rather than replacing theoretical learning.

(last update: 2026-02-17 02:02)

Prerequisites

To use this cookbook effectively, you need:

  • Cluster: OpenShift 4.18+ with the OpenShift Virtualization operator installed.

  • Tools: oc, kubectl, and virtctl. (Optional: kustomize, helm, ansible).

  • Knowledge: Proficiency with YAML, and CLI workflows (not mandatory but strongly suggested). Basic understanding of Kubernetes/OpenShift concepts.

Go through all the Getting Started modules, especially if you are new to the technology, to prepare or verify your environment.

Cookbook Modules

Select one of the module from the left menu to begin:

  • Getting Started - Essential tools setup and environment verification.

  • Storage Configuration - Storage setup for virtual machines.

  • Networking Configuration - Networking guides.

  • Virtual Machine Configuration - Cloud-init and VM customization.

  • YAML Manifests Library - A library of ready-to-deploy manifest examples.

  • API Integration - API components architecture and functions such as cloning, export and machine topology.
